# HIVE Roadmap

HIVE - Hyper Intelligence for Verified Engineering - is built incrementally. Core orchestration and execution loop components are heavily tested and **verified**, while advanced features (like parallel swarms and experimental handoffs) remain in active development.

## Completed Phases (Verified Core)

- **Phase 1:** Verified Patch Runner Foundation
- **Phase 1.5:** Durable Task Storage + Dashboard Surfacing
- **Phase 2:** Multi-Agent Council Execution
- **Phase 2.5:** Agentic Patch Guardrails
- **Phase 3:** GitHub PR Mode
- **Phase 3 Hardening:** Explicit confirmation gates + negative-path tests
- **Phase 3.5:** CLI Workflow
- **Phase 4A:** Web-based Editor Client
- **Phase 4.1:** Editor Client Validation

*Note: The features in the completed phases above make up the Verified Core and have passed rigorous automated testing for safety, isolation, and explicit human-in-the-loop validation.*

## Experimental & Underlying Platform Phases

The following features belong to the underlying Hive Mind legacy platform and are actively being stabilized or ported into the verified HIVE loop:

- **Phase 5:** Auth, Queues & Deployment (BullMQ, NextAuth)
- **Phase 6:** Grok (xAI) Integration
- **Phase 7:** Swarm, Bots & Telemetry (Parallel execution)
- **Phase 8:** Search, Synthesis & Scheduler

## Next Phases (Upcoming)

- **Phase 5 (HIVE):** Public Demo and GitHub Launch (Current Phase)
- **Phase 6 (HIVE):** Advanced Context Assembly (Integrating RAG directly into the Builder prompt)
- **Phase 7 (HIVE):** Multi-File Diff Optimization
- **Phase 8 (HIVE):** IDE Extension (VS Code / JetBrains integration of the Editor Client)
